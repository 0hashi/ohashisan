Domain Name SystemDNS
 UDP/TCP 53 Converts hostnames/FQDNs to IP addresses. Also can convert IP addresses to FQDNs if reverse lookup has been configured. Large DNS transfers may use TCP.
Dynamic Host Configuration ProtocolDHCP
 UDP 67 & 68 Automatically assigns IP addresses and other configuration options like: subnet mask, default gateway, DNS settings, etc. Receiving a DHCP address is a four step process. Think, DORA (Discover, Offer, Request, Acknowledge).
Internet Message Access Protocol v4IMAP4
 TCP 143 - Clear
TCP 993 - TLS		 Includes management of email inbox from multiple clients. If one client deletes an email, it's deleted on the server and removed from other client inboxes.
Post Office Protocol v3POP3
 TCP 110 - Clear
TCP 995 - TLS		 Post Office Protocol v3 - Other protocols used for clients to receive email.
Simple Mail Transfer ProtocolSMTP
 TCP 25 - Clear
TCP 587 - TLS		 SMTP is used to send email, IMAP4 and POP3 are protocols used to access and manage email on remote SMTP servers.
Trivial File Transfer ProtocolTFTP
 UDP 69 Simple file transfer protocol, reads and writes files, no authentication, no encryption, not used on production systems.
Simple Network Management ProtocolSNMP v1, 2 & 3 SNMPTRAP
 UDP 161
UDP 162		 SNMP v3 is the latest and greatest version providing message integrity, authentication and encryption.
SNMP Trap
Network Time ProtocolNTP
 UDP 123 NTP ensures every device on the network has the same time. In most cases it's best to use pool.ntp.org. Synchronizing the clocks on system is crucial to ensuring systems and applications work as expected.
Session Initiation ProtocolSIP
 TCP 5060 & 5061 Voice over IP (VoIP) signaling. Used to set up and manage VoIP sessions (call, ring, hang up). Is also used for video conferencing, instant messaging, file transfer and more...
Server Message Block (CIFS)SMB
 TCP 445 Used for file and print sharing, AKA CIFS (Common Internet File System).
Internet Control Message Protocol ICMP
 The Internet Control Message Protocol (ICMP) does not have ports, but instead uses types and codes. ICMP is an Internet Layer protocol, so it doesn't have an associated port number like application level protocols that are built on top of the transport layer.
